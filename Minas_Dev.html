  • HTML

    • A história do html se inícia em 1991, ela foi criada por Tim Berners-Lee, no CERN (European Council for Nuclear Research) na suíça. Inicialmente o HTML foi projetado para interligar instituições de pesquisa próximas, e compartilhar documentos com facilidade. Em 1992, foi liberada a biblioteca de desenvolvimento WWW ( World Wide Web), uma rede de alcance mundial que junto com o HTML proporcionou o uso em escala mundial da web.

    O Html é uma linguagem de marcação e não linguagem de programação como muitos pensam, quando você está programando em html não significa que está programando para fazer alguma coisa automatica e sim está demarcando aquele espaço com as tags como header, body e outras essas tag são seguidas de atributos chamados class ou id que são identificadores daquele espaço que está usando e depois você irá estilizar essa tag na qual demarcou.Vamos conversar sobre estilizar na seção de css ok?

    Os atibutos chamados id tem de ter um valor único na pagina toda ou seja você pode usar a identificação que colocou uma única vez como exemplo vamos pegar um header e nele foi usado a id ="valor" você fez as estizações dele e gostou e quer fazer o footer igual mas não pode repetir a id o que fazer?, nesses casos usamos o atributo de nome class com ele é possível usar o mesmo valor de identificaçação em varias tags sem problemas Abaixo uma figura de como tem de ser uma estrura Html e vamos falar o que significa cada um.

    Esta é a estrutura básica necessária para iniciar um desenvolvimento em html

    DOCTYPE:essa linha quer dizer qual tipo de html você está usando e esse é o 5, teve outros mas esse é o mais atualizado e importande de se estudar pois ele tem tags novas que nos antigos estão em desuso o chamado deprecated que quer dizer descontinuado

    html lang="en"diz qual idioma está escrito en de inglês e pt-br de português do Brasil e tem outros idiomas que tem de definir conforme o local para qual é, afinal o browser precisa de uma forma de identificar de onde é então usamos essa.

    head:aqui é onde colocamos todas as outras tags importante para que o browser identifique o documento. O charset UTF-8 é o que usamos na web atual e faz parte de um padrão chamado Unicode. A ideia é que com ele possamos representar qualquer caractere de qualquer idioma. Ao usar o charset dentro da tag meta de nosso arquivo estamos dizendo ao navegador, quando for ler esse arquivo use as regras do charset UTF-8

    meta http-equiv="X-UA-Compatible" content="IE=edge": A meta tag X-UA-Compatible permite que os autores da web escolham a versão do Internet Explorer na qual a página deve ser renderizada.

    meta name="viewport" content="width=device-width, initial-scale=1.0: esta tag é umas da mais importantes pra responsividade da tela, se não colocar sua pagina irá apenas aumentar e diminuir o zoom, com ela você poderá fazer funcionar conforme as queries que foram definidas quando quando foi estilizada para o tamanho de tela escolhida, o que são queries?, é uma forma de também estilizar a pagina conforme o tamanho do dispositivo.

    title: Aqui irá colocar o nome do titulo do seu texto, ele será o que vai aparecer no navegador.

    body: Nesta irá colocar todas tags que terá os conteúdos e que vai mostrar no navegador, utilizada dentro do body para dividir os conteúdos que são as descritas abaixo

    section: Esta é uma tag do html5 assim como as outras abaixo, ela ira dividir seus conteúdos, dentro dela poderá por outras que irão se tornar filhas, netas e as que ficarem no nivel da tag colocada serão irmãs e assim continua.

    main: Aqui quer dizer principal então cuide bem onde quer colocar essa tag, pois irá mostrar as coisa que é considerada principal no navegador quando for pesquisado algo relacionado com o que queira. O main não pode ser filho de uma section mas o inverso pode.>

    article: O article é uma seção de atigos onde irá escrever aquilo na qual refere seguindo toda uma estrutura de h1 como titulo depois usará os paragrafos, você pode usar quantos for preciso. Cuidado quando for escolher entre article e section pois os dois nos confunde na hora de usarmos ou um ou outro porque ambos usamos para separar por seções mas semanticamente use o article no caso de algo mais objetivo.
